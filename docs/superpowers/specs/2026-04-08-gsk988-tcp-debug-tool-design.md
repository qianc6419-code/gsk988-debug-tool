# GSK988 以太网调试工具 — 设计文档

**版本：** 1.0
**日期：** 2026-04-08
**技术栈：** C++ + Qt 5.15.x LTS（静态编译）
**兼容性：** Windows 7 / Server 2016 及以上

---

## 1. 项目概述

本工具用于 GSK988 系列数控系统的以太网 TCP 通讯调试与对接。

### 1.1 核心目标

- 提供可视化界面与 GSK988 设备进行 TCP 通讯
- 支持真实设备连接和本地 Mock 服务器两种运行模式
- 无真实设备时，Mock 服务器可配置响应数据，用于脱机测试
- 完整实现 GSK988 通讯协议的所有命令类型

### 1.2 运行环境

- **编译目标：** Windows x64 静态编译，单文件 exe，无需运行时依赖
- **最低系统：** Windows 7 / Windows Server 2016
- **Qt 版本：** 5.15.x LTS（静态编译商业版）

## 3. Tab 功能详细设计

### 3.1 实时数据 Tab

展示设备当前状态的数据面板。

**布局：** 2×2 网格卡片

| 卡片 | 内容 |
|------|------|
| 设备状态 | 运行状态、模式、进给倍率、主轴转速 |
| 机械坐标 | X/Y/Z 轴机械坐标，实时更新 |
| 工件坐标 | X/Y/Z 轴工件坐标 |
| 当前报警 | 报警号、报警内容、报警时间 |

**下方：** 原始 HEX 数据显示框（只读，等宽字体着色）
**刷新：** 手动刷新（快捷按钮）+ 自动轮询（可配置间隔 1-10秒）

### 3.2 发送指令 Tab

预定义命令表格，完整协议覆盖。

**筛选栏：**
- 搜索框（按命令名、命令码模糊搜索）
- 类型下拉：全部 / 读命令 / 写命令 / 控制命令
- 功能下拉：全部 / 设备信息 / 状态坐标 / 报警诊断 / 参数操作 / 运行控制 / PLC / 诊断

**命令表格列：**

| 列 | 说明 |
|----|------|
| 类型 | 读(绿)/写(橙)/控制(红) 标签 |
| 命令码 | 十六进制，如 01H |
| 名称 | 命令中文名称 |
| 说明 | 一句话描述 |
| 快捷参数 | 内嵌输入框/下拉框（如参数号下拉、控制类型下拉） |
| 操作 | [发送] 按钮 |

**支持的完整命令列表：**

**读命令：**
- 01H 读取设备信息
- 02H 读取机床状态
- 03H 读取坐标数据
- 04H 读取PLC状态
- 05H 读取报警信息
- 06H 读取参数
- 07H 读取诊断信息
- 08H-0FH 扩展读命令（预留）
- 10H 读取数据块

**写命令：**
- 11H 写参数
- 12H-1FH 扩展写命令（预留）
- 20H 控制命令（进给保持、复位、急停等）
- 21H 写PLC
- 22H 写数据块

**控制命令（30H系列）：**
- 30H 启动
- 31H 停止
- 32H 进给保持
- 33H 复位
- 34H 急停
- 35H-3FH 扩展控制命令

**数据块传输（50H系列）：**
- 50H 读取数据块
- 51H 写数据块

**发送流程：**
1. 用户选择参数（可选）
2. 点击发送
3. 构建协议帧（加XOR校验 + CRC16）
4. 发送并等待响应
5. 解析响应，更新实时数据Tab和日志

### 3.3 数据解析 Tab

手动输入 HEX 数据进行解析验证。

**输入区：** HEX 文本框（粘贴完整协议帧）
**输出区：** 结构化解析结果（帧头/命令码/数据长度/数据内容/XOR/CRC）

### 3.4 通讯日志 Tab

实时显示所有发送和接收的通讯记录。

**日志格式：**
```
[时间] [方向] [命令码] [HEX原始数据]
  └─ 解析结果
```

**筛选：** 全部 / 发送 / 接收 / 错误
**操作：**
- 清空日志（内存）
- 保存本次会话 → 导出到本地文件（格式可选：TXT / CSV）

### 3.5 参数管理 Tab

读取和修改CNC系统参数。

**左侧：** 参数号树形列表（按类别分组）
**右侧：** 参数详情（参数号、名称、当前值、最小/最大值、描述）
**操作：** 读取选中参数 / 写入新值 / 批量读取

## 4. Mock 服务器设计

### 4.1 概述

当运行模式切换为"Mock服务器"时，程序在本地 TCP 端口 8067 启动一个模拟 GSK988 设备响应的服务器。

上位机程序连接 `127.0.0.1:8067` 即可与 Mock 服务器通讯。

### 4.2 可配置响应

每个命令码对应一个可配置的响应数据模板，存储在内存中。

**配置界面：** 独立的"Mock配置"面板（可在侧边栏展开或独立Tab）
- 下拉选择命令码
- HEX 编辑器编辑响应数据体
- [应用] [重置默认] 按钮

**默认响应：** 预设合理的固定数据（如设备信息返回固定型号、坐标返回合理范围内的随机值）

### 4.3 数据变化策略

- 坐标数据：每次响应时在基准值上 ± 小幅随机变化（模拟设备运动）
- 状态数据：固定返回"运行中"状态
- 报警数据：默认无报警
- 参数数据：可配置

## 5. 协议实现

### 5.1 帧结构

```
+---------------+---------------+--------+--------+--------+--------+--------+--------+--------+
|   帧头 (2B)   |   长度 (2B)   |  CMD   |  SUB   |  Type  |  Data  | XOR    |  CRC16 (2B)|
|   5A A5       |   00 7E       |  01H   |  00    |  00    |  ...   |  --    |  ----     |
+---------------+---------------+--------+--------+--------+--------+--------+--------+--------+
```

### 5.2 关键实现点

- **字节序：** 小端序（Little Endian）
- **XOR校验：** 数据区所有字节异或，结果取低8位
- **CRC16：** MODBUS RTU CRC（多项式 0x8005）
- **超时处理：** 发送后 5 秒无响应提示超时
- **断线重连：** 检测到断线后自动重连，最多重试 3 次

## 8. 验收标准

1. 启动后在无网络的环境下可以切换到 Mock 模式并正常收发数据
2. 真实设备模式下，正确 IP/端口可建立连接，断线可检测
3. 所有 28+ 条命令均可正确发送，响应数据可正确解析
4. 发送帧的 XOR 和 CRC16 校验正确
5. 通讯日志实时显示发送/接收的 HEX 数据
6. 手动保存可将当前日志导出为文件
7. 静态编译后的单个 .exe 文件可在 Windows 7/Server 2016 上直接运行
