# Fanuc FOCAS 协议插件设计

## 概述

将 Fanuc CNC 以太网数据采集协议集成为多协议调试工具的第三个协议插件，遵循现有的 IProtocol + IProtocolWidgetFactory + ITransport 架构，与 GSK988、Modbus TCP 并列。

## 协议帧结构

### 请求帧

```
┌──────────┬──────────┬──────────┬──────────────┬──────────────────┐
│ 协议头    │ 功能码    │ 数据长度  │ 数据块个数    │ 数据块 × N       │
│ 4B       │ 4B       │ 2B       │ 2B           │ 每块 28B         │
│ A0×4     │ 0001     │          │              │                  │
│          │ 2101     │          │              │                  │
└──────────┴──────────┴──────────┴──────────────┴──────────────────┘
```

单数据块 (28B): 块长度(2B) + NC/PMC(2B) + 预留(2B) + 块功能码(2B) + 参数区(20B)

### 握手帧

连接后必须先发送握手帧：
`A0 A0 A0 A0 | 00 01 | 01 01 | 00 02 | 00 02`

### 数值解析

响应数据中的数值用 `calcValue` 解析：分子(int32, 大端) / 分母底(short)^指数(short)

## 命令定义

按块功能码分为以下几类：

### 系统信息
- `0x18` — 系统信息（NC型号、设备类型）

### 运行状态
- `0x19` — 运行信息（操作模式、运行状态、急停、告警状态）

### 主轴
- `0x25` — 主轴速度
- `0x40` — 主轴负载
- PMC读 G30 — 主轴倍率
- PMC读 F22-F25 — 主轴速度设定值

### 进给
- `0x24` — 进给速度
- PMC读 G12 — 进给倍率
- 宏变量读 4109 — 进给速度设定值

### 计数与时间
- 宏变量读 3901 — 工件数
- 参数读 6751+6752 — 运行时间（毫秒+分钟）
- 参数读 6753+6754 — 加工时间
- 参数读 6757+6758 — 循环时间
- 参数读 6750 — 上电时间

### 程序与刀具
- `0x1C` — 程序号
- 宏变量读 4120 — 刀具号

### 告警
- `0x23` — 告警信息列表

### 坐标
- `0x26` — 坐标（type: 0=绝对, 1=机械, 2=相对, 3=剩余距离）

### 宏变量
- `0x15` — 读宏变量
- `0x16` — 写宏变量

### PMC
- `0x8001` — 读PMC（支持 G/F/Y/X/A/R/T/K/C/D 地址类型，byte/word/long/float 数据类型）
- `0x8002` — 写PMC

### 参数
- `0x8D` — 读参数

## 实时数据页面设计

全量轮询所有数据项，连接并握手成功后自动开始循环请求。

轮询顺序：
1. 系统信息 → 运行信息 → 主轴速度 → 主轴负载 → 主轴倍率 → 主轴速度设定值
2. 进给速度 → 进给倍率 → 进给速度设定值 → 工件数
3. 运行时间 → 加工时间 → 循环时间 → 上电时间 → 程序号 → 刀具号
4. 告警 → 绝对坐标 → 机械坐标 → 相对坐标 → 剩余距离坐标
5. 回到步骤1

UI 布局：
- 上方：轮询配置区（自动刷新开关、间隔选择、手动刷新按钮）
- 中间：数据表格，按类别分组显示
  - 系统信息组：NC型号、设备类型
  - 运行状态组：操作模式、运行状态、急停状态、告警状态
  - 主轴组：速度、负载、倍率、速度设定值
  - 进给组：速度、倍率、速度设定值
  - 计数组：工件数
  - 时间组：运行时间、加工时间、循环时间、上电时间
  - 程序组：程序号、刀具号
  - 告警组：告警列表
  - 坐标组：绝对/机械/相对/剩余（各XYZ三轴）
- 下方：Hex 收发显示

| 计数 | 工件数 |
| 时间 | 运行时间、加工时间、循环时间、上电时间 |

## 数据解析页面设计

与 Modbus 的 ParseWidget 一致：手动输入 Hex 帧进行解析，显示帧结构和各字段含义。

## Mock 服务器

MockServer 根据 Fanuc 协议生成模拟响应：
- 系统信息返回固定 NC 型号 "31" 和设备类型 "0B"
- 运行信息返回 "MEM" 模式 + "STOP" 状态
- 主轴/进给速度返回模拟数值（如 1500 RPM、500 mm/min）
- 坐标返回模拟 XYZ 值
- 告警返回空列表
- PMC/宏变量维护内存映射，支持读写

## MainWindow 集成

- 协议下拉框增加 "Fanuc FOCAS" 选项（index 2）
- `switchProtocol()` 中增加 `case 2` 创建 `FanucProtocol` + `FanucWidgetFactory`
- 连接后自动发送握手帧，握手响应后启动轮询
- 切换协议时默认显示实时数据页面（已有逻辑）

## 数据类型

Fanuc 响应中的数值编码：
- int32（4字节大端）：分子、整型参数值
- short（2字节大端）：分母底数、分母指数、short 值
- 字符串：NC 型号、告警消息

## 帧校验

接收端 `checkFrame()` 校验：
- 协议头必须是 `A0 A0 A0 A0`
- 功能码位置必须是 `00 01 21 02`（响应的功能码 `2102` 与请求的 `2101` 不同）

## 提取帧

`extractFrame()` 从 buffer 中提取完整帧：
- 读取偏移量 8-9 的数据长度字段（2字节大端）
- 总帧长 = 10 + 数据长度
