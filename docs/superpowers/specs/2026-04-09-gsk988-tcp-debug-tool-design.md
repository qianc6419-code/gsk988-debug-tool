**版本：** 2.0
**日期：** 2026-04-09
**技术栈：** C++ / Qt 5.15.x LTS
**范围：** 只实现读取类命令，不含写入/控制操作

## 1. 项目概述

重新构建 GSK988 系列数控系统的以太网 TCP 通讯调试工具。支持真实设备连接和内置 Mock 服务器两种运行模式，无真实设备时可脱机测试。

### 1.1 核心目标

- 可视化界面与 GSK988 设备进行 TCP 通讯调试
- 完整实现协议文档中所有读取类命令
- 内置 Mock 服务器，内置默认响应数据，无需配置即可脱机使用
- 最终成品可直接对接真实设备

### 1.2 运行环境

- Qt 5.15.x LTS
- Windows 平台

### 2.3 模块职责

- **Gsk988Protocol** — 知道所有命令定义、帧格式、校验算法。提供 `buildRequest(cmdCode, params)` 和 `parseResponse(data)` 接口
- **FrameBuilder** — 底层帧拼接、长度计算、帧头帧尾封装
- **TcpClient** — 管理 TCP 连接，发送请求帧，接收响应帧，通过信号通知上层
- **MockServer** — QTcpServer 子类，监听本地端口，收到请求后用内置默认数据生成响应帧
- **MainWindow** — 组装工具栏和 Tab 页，协调 Network 和 UI 之间的数据流

## 3. 协议层设计

### 3.1 帧格式

```
请求帧: 0x93 0x00 | 长度(2B) | 请求标识(8B) | 数据域 | 0x55 0xAA
响应帧: 0x93 0x00 | 长度(2B) | 应答标识(8B) | 数据域 | 0x55 0xAA

数据域: 命令码(1B) + 错误码(2B) + [附加数据(变长)]
长度 = 数据域总字节数（不含帧头0x9300和帧尾0x55AA）
```

请求标识固定: `0x6F 0xC8 0x1E 0x64 0x1E 0x17 0x10 0x17`
应答标识固定: `0x00 0x64 0x1E 0xC8 0x1E 0x17 0x10 0x01`

### 3.2 Gsk988Protocol 核心接口

```cpp
class Gsk988Protocol : public QObject {
    // 构建请求帧
    QByteArray buildRequest(quint8 cmdCode, const QByteArray& params = {});

    // 解析响应帧
    struct ParsedResponse {
        quint8 cmdCode;
        quint16 errorCode;
        QByteArray data;    // 原始附加数据
        bool isValid;
    };
    ParsedResponse parseResponse(const QByteArray& frame);

    // 校验
    bool validateFrame(const QByteArray& frame);
};
```

## 4. 网络层设计

### 4.1 TcpClient

```cpp
class TcpClient : public QObject {
    void connectTo(const QString& ip, quint16 port);  // 默认端口6000
    void disconnect();
    bool isConnected() const;
    void sendFrame(const QByteArray& frame);

signals:
    void connected();
    void disconnected();
    void connectionError(const QString& msg);
    void responseReceived(const QByteArray& frame);
};
```

- 内部使用 `QTcpSocket`，异步非阻塞
- 接收数据时做帧边界检测：找 `0x93 0x00` 帧头，按长度字段读取完整帧，校验 `0x55 0xAA` 帧尾
- 断线自动检测，发出 `disconnected` 信号
- 超时机制：发送后 3 秒无响应视为超时

### 4.2 MockServer

```cpp
class MockServer : public QObject {
    bool start(quint16 port = 6000);
    void stop();
    bool isRunning() const;

signals:
    void logMessage(const QString& msg);
};
```

- 内嵌 `QTcpServer`，启动后监听指定端口
- 收到请求帧 → 调用 Gsk988Protocol 解析命令码 → 查内置数据表生成响应帧 → 发回
- 内置合理默认数据，无需配置即可使用

### 4.3 Mock 内置默认数据

| 命令 | Mock 返回值 |
|------|------------|
| 0x0A | 错误码=0, 权限=2（可读写） |
| 0x0B | "GSK988TD" |
| 0x10 | 模式=3（自动） |
| 0x11 | 状态=1（运行中） |
| 0x21 | X=123.456, Y=-45.678, Z=0.000 |
| 0x17 | 刀号=1, 刀补号=1 |
| 0x16 | 件数=42 |
| 0x81 | 错误数=0, 警告数=0 |
| 其他 | 错误码=0 + 合理默认值（零值或空） |

### 4.4 连接协调逻辑

MainWindow 统一管理两种模式：

- **真实设备模式：** TcpClient.connectTo(ip, port)
- **Mock 模式：** MockServer.start() + TcpClient.connectTo("127.0.0.1", port)

两者统一通过 TcpClient 的 `responseReceived` 信号接收数据，上层不关心数据来源。

### 5.2 Tab 1：实时数据

2×2 卡片布局，自动轮询展示设备状态：

- **设备状态卡片：** 运行模式、运行状态、进给倍率、主轴转速
- **机械坐标卡片：** X/Y/Z 轴坐标，实时更新
- **加工信息卡片：** 加工件数、运行时间、切削时间、当前刀号

### 5.3 Tab 2：发送指令

所有读取命令的表格，支持筛选和发送：

- **筛选栏：** 搜索框（按命令名/命令码模糊搜索）+ 类别下拉（全部/通讯管理/实时信息/参数/刀补/宏变量/丝杠/轴IO/报警诊断/其他）
- **表格列：** 类型标签(颜色) | 命令码 | 名称 | 通道输入 | 参数输入 | 发送按钮
- 需要通道号的命令显示通道 SpinBox（默认 0）
- 需要参数号的命令显示参数输入框
- 发送后下方实时显示解析结果

### 5.4 Tab 3：数据解析

手动输入 HEX 帧进行解析验证：

- **输入区：** HEX 文本框（粘贴完整协议帧），支持自动忽略空格、逗号等分隔符
- **解析区：** 逐字段着色显示：帧头、帧长度、标识、命令码、错误码、附加数据、帧尾
- 自动识别请求帧/响应帧
- 根据命令码做语义解析（如权限=2表示可读可写）

### 5.5 Tab 4：通讯日志

实时记录所有收发数据：

- **日志格式：** `[时间] [方向] [命令码] [描述]` + 下一行 HEX 原始数据
- **筛选：** 全部 / 发送 / 接收 / 错误
- **操作：** 清空日志、导出为 TXT 文件

## 6. 错误处理

- TCP 连接失败/断线：状态指示变红，日志记录错误信息，自动停止轮询
- 响应超时（3秒）：日志记录超时警告，不重试（由用户决定）
- 协议帧校验失败：日志记录帧格式错误，显示原始 HEX
- 命令错误码非零：在发送结果区域显示错误码含义
